## 术语（Terminology）

为了设计类似 S3 的对象存储，我们需要先了解一些核心的对象存储概念。本节介绍了对象存储中使用的相关术语。

- Bucket（桶）：用于存放对象的逻辑容器。桶名称在全局范围内必须唯一。要将数据上传到 S3，首先必须创建一个桶。

- Object（对象）：对象是我们存储在桶中的独立数据单元。它包含对象数据（也称为 payload）和元数据。对象数据可以是我们想存储的任意字节序列。元数据是一组描述该对象的键值对。

- Versioning（版本控制）：一种使同一桶中对象能够保存多个版本的功能。版本控制在桶级别启用。这一特性可以帮助用户恢复那些因误操作被删除或覆盖的对象。

- Uniform Resource Identifier (URI)（统一资源标识符）：对象存储提供 RESTful API 来访问其资源，包括桶和对象。每个资源都可以通过其 URI 唯一标识。

- Service-level agreement (SLA)（服务等级协议）：服务等级协议是服务提供商与客户之间订立的合同，定义了服务可用性和性能保证。例如，Amazon S3 的低频存储（Standard-Infrequent Access）提供了如下 SLA [7]：
  - 设计目标是在多个可用区（Availability Zones）之间实现 99.999999999% 的对象持久性。
  - 即使整个可用区完全毁坏，数据依然保持可靠可用。
  - 设计目标是 99.9% 的可用性。
